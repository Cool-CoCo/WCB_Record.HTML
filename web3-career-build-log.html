📖 学习内容
一个完整的 Web3 产品或项目通常需要三类 Builder 共同协作：
Research

发现问题，理解协议、产品、治理、生态机会和风险
Ops

连接用户，设计社区运营、Space / 活动策划、传播执行和反馈复盘
Dev

解决问题，把需求变成可运行的原型、工具、合约、脚本或 Agent 能力

DAY 2 路径启动：把职业方向变成可执行 Scope
🎯 学习目标
  • Research 学习如何提出一个好的 Research Question，明确资料边界
  • Ops 学习如何定义一场基础 Space / AMA / 小型线上活动
  • Dev 学习如何定义一个 AI-assisted MVP，控制开发 Scope
📖 学习内容
  • • Research Track 采用"两条线"：一个任务专门读 EIP / MIP，其他任务研究真实产品 / 协议
  • • 从 DefiLlama Top TVL 协议、Uniswap / Morpho / Pendle 等选择研究对象
  • • 写清研究问题、资料来源、产品分类、核心机制和想回答的用户 / 产品问题
  • • 区分事实、数据、假设、推断和个人判断
Day 2 的 Ops 重点是"把一个活动想清楚"，需要定义：
  • • 活动目标：为什么要做这场 Space？
  • • 目标听众：面向新人、开发者还是研究者？
  • • 核心主题：一句话说清，避免标题很大但讨论发散
  • • 关键动作：希望听众完成什么动作？
  • • 基础闭环：预热 → 参与 → 互动 → 反馈 → 复盘
  • • 定义一个 AI-assisted Web3 Prototype（自己设计或参与 Moss Open Source Challenge）
  • • 写清 Scope、Architecture、Tech Stack、真实实现 / mock / 本周不做的边界
